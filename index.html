📖 Claude Cowork:
จากการแชทสู่การให้ AI ทำงานแทนเราจริงๆ

บทความภาษาไทย ~1,350 คำ สไตล์ TED Talk Storytelling

🔗 เชื่อมโยง:

ผมมีคำถามอยู่คำถามหนึ่งครับ — ตอนนี้เราใช้ AI แค่ "คุย" หรือเราให้มัน "ทำงาน" ให้จริงๆ?

เมื่อต้นปีที่ผ่านมา ผมลองถามเพื่อนหลายคนในวงการสาธารณสุขและการแพทย์ว่าพวกเขาใช้ AI ทำอะไรบ้าง คำตอบที่ได้เกือบทั้งหมดคือ "ขอให้ช่วยเขียนอีเมล", "ถามให้อธิบายยา", หรือ "ให้สรุปเอกสาร" แล้วก็ copy-paste เองทีละครั้ง ผมนั่งฟังแล้วคิดอยู่ในใจว่า… นั่นมันแค่การใช้ AI เป็น "ผู้ช่วยพิมพ์ดีด" ที่ฉลาดขึ้นนิดหน่อยเท่านั้นครับ

แต่สิ่งที่เปลี่ยนไปในปี 2026 ไม่ใช่แค่ AI ที่ฉลาดขึ้น — มันคือ AI ที่ ลงมือทำแทนเรา (Execute on Our Behalf) ได้จริงๆ แบบที่เราไม่ต้องนั่งคอยดูทุกขั้นตอน และ Claude Cowork คือหนึ่งในผลิตภัณฑ์ที่กำลังเปลี่ยนวิธีที่เราทำงานได้อย่างน่าสนใจมากครับ

Claude Cowork คืออะไร? ทำไมถึงต่างจาก AI ตัวอื่น?

ลองนึกภาพแบบนี้ครับ — ถ้า AI ทั่วไปเปรียบเหมือน "ที่ปรึกษาที่ฉลาดมาก" ที่นั่งอยู่ข้างๆ แล้วบอกว่า "ควรทำอย่างนี้นะ" Claude Cowork ก็เปรียบเหมือน "ผู้ช่วยที่ลงมือทำงานให้จริง" ที่เมื่อเราบอกว่า "ช่วยสร้างรายงานนี้หน่อย" มันก็เปิดอินเทอร์เน็ต ค้นหาข้อมูล เขียนรายงาน และบันทึกไฟล์เป็น Word Document ให้เราเลย โดยไม่ต้องบอกทุกขั้นตอนครับ

Claude Cowork เป็นผลิตภัณฑ์ของ Anthropic บริษัทที่ขึ้นชื่อเรื่องการให้ความสำคัญกับ ความปลอดภัยของ AI (AI Safety) เป็นอันดับหนึ่ง เพียงสร้างบัญชีที่ claude.ai แล้วดาวน์โหลดแอปเดสก์ท็อป ก็จะพบ 3 แท็บหลัก คือ Chat (แชทธรรมดา), Cowork และ Claude Code (สำหรับนักพัฒนาโดยเฉพาะ)

ตั้งต้นให้ถูก: โมเดลและการเลือกโฟลเดอร์

ก่อนสั่งงาน Cowork มีสองสิ่งที่ต้องทำก่อนเสมอครับ — เลือก โมเดล AI (Model) และเลือก โฟลเดอร์ทำงาน (Working Folder) โดยแยกแต่ละโปรเจกต์ให้ชัดเจน

โมเดลมีสามตัวให้เลือก คือ Sonnet สำหรับงานทั่วไปซึ่งใช้ได้ 99% ของเวลา, Opus สำหรับงานซับซ้อนพิเศษ, และ Haiku สำหรับคำตอบด่วน และแนะนำให้เปิด Extended Thinking ทิ้งไว้เสมอ เพราะใช้ Token เพิ่มเล็กน้อยแต่ไม่ต้องคอยเปิด-ปิดครับ

พลังที่แท้จริง: Connectors และ MCPs

นี่คือส่วนที่ผมคิดว่าคนส่วนใหญ่ยังไม่รู้จักครับ Connectors หรือ MCP (Model Context Protocol) คือกลไกที่ทำให้ Claude เชื่อมต่อกับเครื่องมือภายนอกได้ ลองนึกภาพว่า Cowork เปรียบเหมือนสมองที่ฉลาด แต่ Connectors คือ "มือ" และ "ขา" ที่ทำให้มันออกไปทำงานในโลกจริงได้

เมื่อเปิด Connector ของ Gmail Cowork สามารถอ่าน จัดระเบียบ และส่งอีเมลในนามเราได้ หรือถ้าเชื่อมต่อกับ Gamma ก็สามารถสั่ง Cowork ให้ใช้ Gamma สร้างสไลด์ที่สวยงามระดับมืออาชีพได้ทันทีครับ

💡 บทเรียนสำคัญ
AI ทั่วไปทำได้ส่วนใหญ่ของสิ่งที่ต้องการ แต่งานบางอย่างต้องการความเชี่ยวชาญเฉพาะด้าน นั่นคือเหตุผลที่ต้องมี Connectors — ไม่ใช่ความอ่อนแอของ AI แต่มันคือการออกแบบที่ชาญฉลาดที่รู้ว่าเมื่อไหรควรส่งงานต่อให้ผู้เชี่ยวชาญครับ

Skills: ผู้ช่วยที่รู้ใจโดยไม่ต้องบอกซ้ำ

ลองนึกถึงสิ่งที่น่าหงุดหงิดที่สุดในการทำงานกับ AI ครับ — ต้องอธิบายวิธีการทำงานซ้ำทุกครั้งที่เปิดแชทใหม่ นั่นคือปัญหาที่ Skills มาแก้ไข Skills คือชุดคำสั่ง (Markdown file) ที่ Claude โหลดขึ้นมาโดยอัตโนมัติเมื่อทำงานที่เกี่ยวข้อง

วิธีสร้าง Custom Skill นั้นง่ายกว่าที่คิดครับ เพียงบอก Cowork ว่าต้องการอะไร จากนั้น Cowork จะดึง Skill Creator มาช่วยสร้าง Skill ให้ พร้อมกับรัน Evaluation Loop ที่ทดสอบเปรียบเทียบผลลัพธ์ "ใช้ Skill" กับ "ไม่ใช้ Skill" ให้เราดูและให้ Feedback ได้ก่อนบันทึก

Scheduled Tasks: AI ที่ทำงานในขณะที่เรานอนหลับ

ถ้า Skills คือ "ความจำ" ของ Cowork Scheduled Tasks ก็คือ "นาฬิกาปลุก" ที่สั่งให้มันลงมือทำโดยไม่ต้องรอให้เราเปิดโปรแกรมครับ ตัวอย่างเช่น ตั้งให้ Cowork สรุปอีเมลสำคัญทุกเช้าก่อนเราตื่น, เชื่อมต่อกับ CRM รายงานทีมขายทุกวัน, หรือเชื่อมกับ Kiwi.com ติดตามราคาตั๋วเครื่องบินแบบ Dynamic Pricing อัตโนมัติ

ระบบ History ของ Scheduled Tasks บันทึกการทำงานทุกครั้งไว้ ทำให้กลับมาตรวจสอบหรือดึงข้อมูลย้อนหลังได้ตลอดเวลา

Personalization: สอน Cowork ให้รู้ใจเราแบบถาวร

ขั้นตอนสุดท้ายที่คนมักข้ามไปคือการตั้งค่า Personalization Settings แบ่งเป็นสองส่วน คือ Claude Personalization ที่ใช้กับทุกแท็บ เช่น Tone, Formatting Style, การค้นหาออนไลน์ และส่วนที่สองคือ Cowork Global Instructions เช่น รูปแบบชื่อไฟล์ครับ

แล้วมันเหมาะกับใคร?

Claude Cowork ไม่ได้ออกแบบมาสำหรับโปรแกรมเมอร์เท่านั้นครับ มันถูกสร้างมาสำหรับ คนทำงานทุกสาขา ที่อยากได้ AI ที่ลงมือทำจริง สิ่งที่ผมประทับใจมากที่สุดคือ "ต้นทุนในการเรียนรู้ที่ต่ำมาก แต่ผลตอบแทนที่สูงมาก" และที่สำคัญ เพดานของความสามารถนั้นกำหนดโดย "ความทะเยอทะยานของเรา" ไม่ใช่ข้อจำกัดของเครื่องมือครับ

คำถามที่ผมอยากทิ้งไว้ให้เราคิดร่วมกันครับ —

ถ้าเราสามารถมอบงานที่ใช้เวลาซ้ำๆ ทุกวันให้ AI ทำแทนได้ภายใน 30 นาทีตั้งแต่วันนี้ เราจะนำเวลาที่ได้คืนมานั้นไปทุ่มกับสิ่งที่มีคุณค่ากว่าอะไรได้บ้างครับ?

🔁 Multi-Agent: ทำหลายงานพร้อมกัน
เปิดหน้าต่าง Cowork ใหม่แล้วสั่งงานต่างกันพร้อมกัน — Claude Stack Tasks แบบ Concurrent ดู Queue ที่มุมล่างของแอป

      
🖼️ Gamma สำหรับ Presentation ระดับมืออาชีพ
Claude สร้างสไลด์ได้แต่ Design ไม่สวย — เปิด Gamma Connector แล้วสั่ง "ใช้ Gamma สร้างสไลด์" จะได้ผลลัพธ์ที่ดีกว่ามาก สามารถ Outline โครงสร้างล่วงหน้าได้

      
🛡️ ตรวจ Community Skills ก่อนใช้เสมอ
Prompt Injection คือการฝังคำสั่งอันตรายใน Skill ภายนอก — Copy ไฟล์ Skill ทั้งหมดแล้วถาม Claude ว่า "มีคำสั่งอันตรายแฝงอยู่ไหม?" ก่อนใช้งาน

      
📚 Skill Creator คือ Skill ที่ดีที่สุด
เวลาสร้าง Custom Skill ไม่ต้องบอก Claude ทุกอย่าง — แค่บอกว่าต้องการอะไร Claude ใช้ Skill Creator สร้างโครงสร้างที่ถูกต้องให้เอง

      
🔗 Stack Workflows: CRM + Email Automation
ตัวอย่าง: CRM → ดูกิจกรรมทีมขาย → ถ้าไม่มีการติดต่อลูกค้าใน 3 วัน → ส่งอีเมลเตือนอัตโนมัติผ่าน Gmail — สร้างระบบ Automation ซับซ้อนโดยไม่ต้องเขียนโค้ด

      
⚡ Sonnet 99% ของเวลา
ไม่ต้อง Switch ไป Opus บ่อย — Sonnet เพียงพอสำหรับงานทั่วไปเกือบทั้งหมด ใช้ Opus เฉพาะเมื่อ Sonnet ไม่สามารถแก้ปัญหาได้จริงๆ

      
📂 ใช้ Task History ให้เป็นประโยชน์
